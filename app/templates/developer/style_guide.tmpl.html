DataTables

For a common data table, we recommend using of Ko-Grid, as it offers best integration with Knockout. We've heavily customized the component to suit our use cases and style, not all functions in the original version are supported and some new features are added.

Basic Usage

To use Ko-grid, you can just need to add the html element in your html page.

<div style="width: 100%; height:300px" data-bind="koGrid: gridOptions"></div>

Then define the "gridOptions" in your view model

self.gridOptions = {
  data: self.userList,
  columnDefs: [{field: 'userID', displayName: 'User ID'},{field: 'firstName', displayName: 'Name'}]
};

Or you can just specify the options in the html:

<div style="width: 100%; height:300px" data-bind="koGrid:  {
  data: self.userList,
  columnDefs: [{field: 'userID', displayName: 'User ID'},{field: 'firstName', displayName: 'Name'}]
}"></div>

"data": the javascript object that provides the datasource for the datagrid. Most often it should be ko.observableArray, so any data change to the datasource will be reflected on the datagrid automatically.

"columnDefs": the javascript literal that used to define the columns of the datagrid

Modes

There are 2 modes to use the customized datagrid in a web page, fixed length or variable length.

Fixed Length

Fixed-length datagrid will have the height of the component set to the height of the HTML element it is inside. The height will not change with the number of rows.

Vertical scrollbar will be enabled when the number of rows exceeds the fixed height. Set option "fixedViewPortHeight" to true (default) to enable this mode, and make sure the height of the div element is set with a value.

<div style="width: 100%; height:300px" data-bind="koGrid: gridOptions"></div>
self.gridOptions = { data: self.userList,
  showFilter: true,
  enablePaging: true,
  fixedViewPortHeight: true,
  columnDefs: [{field: 'userID', displayName: 'User ID'},{field: 'firstName', displayName: 'Name'},{field: 'lastName', displayName: 'Last Name'},{field: 'status', displayName: 'Status'}]
};
Fixed Length Example

Variable Length

Variable-length datagrid will have the height of the component change with the with the number of rows.

There is no Vertical scrollbar. Set option "fixedViewPortHeight" to false to enable this mode, and you don't need to set the height of the div element.

<div style="width: 100%;" data-bind="koGrid: gridOptions"></div>
self.gridOptions = { data: self.userList,
  showFilter: true,
  enablePaging: true,
  fixedViewPortHeight: false,
  columnDefs: [{field: 'userID', displayName: 'User ID'},{field: 'firstName', displayName: 'Name'},{field: 'lastName', displayName: 'Last Name'},{field: 'status', displayName: 'Status'}]
};
Variable Length Example

Paging

Paging is not a must-have in our datagrid. When the number of row is large, developers have 3 options to handle the data:

1. Use fixed-length datagrid without paging.

2. Use fixed-length datagrid with paging.

3. Use variable-length datagrid with paging.

Fixed-length datagrid without paging

In this configuration all the data will be shown in the grid, vertical scrollbar will be enabled if needed. Since ko-grid is not HTML Table based, it can handle tens of thousands of row in the grid.

<div style="width: 100%; height:300px" data-bind="koGrid: gridOptions"></div>
self.gridOptions = { data: self.userList,
  showFilter: true,
  enablePaging: false,
  fixedViewPortHeight: true,
  columnDefs: [{field: 'userID', displayName: 'User ID'},{field: 'firstName', displayName: 'Name'},{field: 'lastName', displayName: 'Last Name'},{field: 'status', displayName: 'Status'}]
};
Fixed-length datagrid without paging Example

Fixed-length datagrid with paging

In this fixed-length configuration, paging is enabled, the number of rows in each page can be customized. They can be very big number, such as 100 or even 500/1000.

To customize the pagesize selection, developer can set the "pagingOption" option like in below.

<div style="width: 100%; height:300px" data-bind="koGrid: gridOptions"></div>
self.gridOptions = { data: self.userList,
  showFilter: true,
  enablePaging: true,
  fixedViewPortHeight: true,
  pagingOptions: {
      pageSizes: ko.observableArray([10, 100, 500, 1000]),
      pageSize: ko.observable(10),
      totalServerItems: ko.observable(0),
      currentPage: ko.observable(1)
  },
  columnDefs: [{field: 'userID', displayName: 'User ID'},{field: 'firstName', displayName: 'Name'},{field: 'lastName', displayName: 'Last Name'},{field: 'status', displayName: 'Status'}]
};
Fixed-length datagrid with paging

Variable-length datagrid with paging

This is probably most common setup in all kinds of datagrid. The grid's height will grow as the number of rows grow. You can also customize the pageSize as needed.

<div style="width: 100%;" data-bind="koGrid: gridOptions"></div>
self.gridOptions = { data: self.userList,
  showFilter: true,
  enablePaging: true,
  fixedViewPortHeight: false,
  pagingOptions: {
      pageSizes: ko.observableArray([10, 100, 500, 1000]),
      pageSize: ko.observable(10),
      totalServerItems: ko.observable(0),
      currentPage: ko.observable(1)
  },
  columnDefs: [{field: 'userID', displayName: 'User ID'},{field: 'firstName', displayName: 'Name'},{field: 'lastName', displayName: 'Last Name'},{field: 'status', displayName: 'Status'}]
};

Row Action

Row action such as "delete", "edit" can be achieved by using custom Celltemplate.

Row Action

We added a "Delete" button for each row in this datagrid.

Notice how the "click" handler is specified and also how the data of clicked row is fetched:

<div style="width: 100%; height:300px" data-bind="koGrid: gridOptionsAction"></div>
self.gridOptionsAction = {
    data: self.userList,
    showFilter: true,
    enablePaging: false,
    fixedViewPortHeight: true,
    columnDefs: [{field: 'userID', displayName: 'User ID'},{field: 'firstName', displayName: 'Name'},{field: 'lastName', displayName: 'Last Name'},{field: 'status', displayName: 'Action', cellTemplate:'<div style="text-align:center"><button data-bind='click: $parent.$grid.$userViewModel.delete.bind($data, $parent.entity.id)'>Delete</button></div>'}]};

self.delete = function(rowid, event) {
    alert('deleting... ' + rowid);
    event.stopPropagation();
}

Format Column Data

There are 3 ways to format column data for display needs. The simplest way is to add computed property to your data object and use that property for display in datagrid.

If adding new property is not possible, you can define a cellFormatter function when defining the column. A limitation of cellFormatter is it will only be provided current cell data to format. Sometimes you may need other data from other columns to do the formatting.

In these cases, you can use "cellTemplate" option in "columnDefs". We will demonstrate here the cellFormatter and cellTemplate usage in following examples.

Format Column Data Using cellFormatter

In this example, we format the Status column from numeric 0 or 1 to text "Disabled" or "Enabled" using a cellFormatter function

<div style="width: 100%; height:300px" data-bind="koGrid: gridOptionsFormat"></div>
self.gridOptionsFormat = {
  data: self.userList,
  showFilter: true,
  enablePaging: false,
  fixedViewPortHeight: true,
  columnDefs: [{field: 'userID', displayName: 'User ID'},{field: 'firstName', displayName: 'Name'},{field: 'lastName', displayName: 'Last Name'},{field: 'status', displayName: 'Status', cellFormatter:function(data){return data == 1 ? 'Enabled' : 'Disabled'}}]
};
Format Column Data Example

Format Column Data Using cellTemplate

The cellTemplate technique for adding row action can be also use for formating column data. See this document for more example of Celltemplate.

In this example, we format the Status column from numeric 0 or 1 to text "Disabled" or "Enabled".

Notice "getProperty($parent)" is used to get the current data of this cell. An Alternative way is "$parent.entity[$data.field]". 

<div style="width: 100%; height:300px" data-bind="koGrid: gridOptionsFormat"></div>
self.gridOptionsFormat = {
  data: self.userList,
  showFilter: true,
  enablePaging: false,
  fixedViewPortHeight: true,
  columnDefs: [{field: 'userID', displayName: 'User ID'},{field: 'firstName', displayName: 'Name'},{field: 'lastName', displayName: 'Last Name'},{field: 'status', displayName: 'Status', cellTemplate:'<span data-bind=\"text: getProperty($parent) != 1 ? \'Disabled\' : \'Enabled\' \"></span>'}]
};
Format Column Data Example

Row Selection

Dates and Time

This page will outline a set of standards that govern the use of date and time information within the browser. The goals for data and time handling are outlined below:

  1. Dates will be formatted for display according to the date format specified in the user's profile.
  2. Times will be formatted for display according to the time format specified in the user's profile.
  3. Time (Date + Time and Time Only) values will be input by the user in the time zone specified in their profile and converted to UTC time for persistence.
  4. Time (Date + Time and Time Only) values will be converted from UTC to the time zone specified in the user's profile before displaying in the browser.
  5. We will provide a Date Picker UI control on any date and/or time input field.

There are three different use cases for which these standards can apply:

  1. Date + Time: The most common use cases includes both a date (year, month, day) and time portion. This is sometimes also represented as a timestamp.
  2. Date Only: Used to record only date information without any notion of time. Examples of this type of date include a person's birthday or an employees start date with the company. Because this data does not have time precision, it MUST NOT be localized.
  3. Time Only: Used to record a particular time of day without a corresponding date element. This is most often used when referring to some recurring time, such as "this employee gets off work at 5:00 every day".

Javascript provides a standard Date type that can be used to represent a date and time value. The Date object's internal representation is epoch time (milliseconds since January 1, 1970 UTC), and can provide a formatted date string in UTC or the local time of the browser. The Date object does not provide a mechanism to convert or display time in any other time zone.

Our project bundles the moment.js and moment-timezone.js libraries to provide additional date/time handling capabilities. See their documentation for a detailed explanation about the capabilities they provide. We have utilized the moment libraries during the implementation of the features below.

Date/Time Input

This section covers the standards for collecting date/time input on your page. There are three different versions of the date picker that correspond to the 3 different use cases outlined above. Your view model MUST declare a dependency on the date-util module in order to use any of these components on your page.

These components are built from the open source bootstrap-datetimepicker project. Custom knockout bindings have been created to simplify the process of using the date picker components. Each of the bindings take 2 parameters:

  1. Main parameter: The name of an observable in your view model. The value of the observable must be a Date or Moment object.
  2. dateOptions (optional): An object whose properties must correspond to one or more of the options described in the bootstrap-datetimepicker documentation.For an example of this, see the Date Picker example below.

Date Picker

The datePicker binding should be used whenever you have a date only input field.

<div class="form-group">
  <label>Date Input Example</label>

  <div class='input-group date' data-bind="datePicker: exampleDate1, dateOptions: {daysOfWeekDisabled: [0,6]}">
    <input class="form-control" placeholder="Choose a Date"> <span class="input-group-addon"><span
      class="glyphicon glyphicon-calendar"></span></span>
  </div>
</div>
Time Picker

The timePicker binding should be used whenever you have a time only input field.

<div class="form-group">
  <label>Time Input Example</label>

  <div class='input-group date' data-bind="timePicker: exampleDate2">
    <input class="form-control" placeholder="Choose a Time"> <span class="input-group-addon"><span
      class="glyphicon glyphicon-calendar"></span></span>
  </div>
</div>
Date + Time Picker

The dateTimePicker binding should be used whenever you have a single input field for both date and time values.

<div class="form-group">
  <label>Date + Time Input Example</label>

  <div class='input-group date' data-bind="dateTimePicker: exampleDate3">
    <input class="form-control" placeholder="Choose a Date and Time"> <span class="input-group-addon"><span
      class="glyphicon glyphicon-calendar"></span></span>
  </div>
</div>

Date/Time Text Formatting

This section covers the standards for outputting date and time values as text on your page. The goal is to display the content according to the user's preferred format (as specified in their profile). A knockout binding handler, dateText, has been created to simplify the process for a developer. This binding handler takes two parameters:

  1. Main parameter: The name of an observable in your view model. The value of the observable must be a Date or Moment object.
  2. include (optional): One of the literal values: date (default), time, or both to determine whether to display the date only, the time only, or the date and time. For an example of this, see the sample code below.

Module "modules/common/dialog-utils" provides functions for displaying various modal alert boxes. To use these functions, just require "modules/common/dialog-utils" in you ViewModel.

alertInfo(message)

dialog.alertInfo('This is a test!');

alertSuccess(message)

dialog.alertSuccess('This is a test!');

alertWarning(message)

dialog.alertWarning('This is a test!');

alertError(message)

dialog.alertError('This is a test!');

confirmationDialog(text, callback, cancelCallback, buttons)

dialog.confirmationDialog('This is a confirmation box, you should say something here.',
                                                                        function(){console.log('OK clicked')},
                                                                        function(){console.log('Cancel clicked')},
                                                                        'Yes|No')

showModalDialog(heading, bodyText, cancelButtonTxt, okButtonTxt, callback, cancelCallback, iconType)

This is a general purpose dialog box, all the above functions are based on this function.

dialog.dialog.showModalDialog('Validation Error',
            'Data validation failed, please see the error message besides the failed input field.',
            null, 'OK', null, null)




Flash Notification

We also provides flash notification functions based on PNotify. This kind of notification is very useful for notifying user certain operation is completed, such as saving editing data. To use these functions, just require "modules/common/dialog-utils" in you ViewModel.

notify(title, message)

dialog.notify('Here goes the title', 'Here goes the message');

notifySuccess(title, message)

dialog.notifySuccess('Here goes the title', 'Here goes the message');

notifyInfo(title, message)

dialog.notifyInfo('Here goes the title', 'Here goes the message');

notifyError(title, message)

dialognotifyError('Here goes the title', 'Here goes the message');




Preloader

Page/data preloader provides an indication for user that some task is running in the background.

Modal Preloader

    <button class="btn btn-primary btn-lg" onclick="$('#modalPreloader').modal('show')"> Launch Modal Preloader </button>
Header Preloader

Header preloader is added to any ajax call by default, there is no need to individually use it in your ajax call.


    <button class="btn btn-primary btn-lg" onclick="$('#loading').show()"> Show Header Preloader </button>
    <button class="btn btn-primary btn-lg" onclick="$('#loading').hide()"> Hide Header Preloader </button>
